# Prime-Token-Protocol-Framework-A-PrimeTalk-and-TRC-Origin

README PTPF — Prime Token Protocol Framework Public Core

## What this is

PTPF Public Core is a portable AI behavior core for passage control, human signal reading, and pragmatic risk filtering.

It is designed to attach to GPTs, agents, workflows, tools, document systems, search systems, and other AI runtime surfaces that can declare input, output, boundary, state, trace, failure, and action limits.

The core file is written for AI runtime interpretation, not for human presentation.

This README is only a human-facing orientation layer.

## What this is not

PTPF Public Core is not a prompt template.
It is not a persona.
It is not a roleplay layer.
It is not an operating system.
It is not a truth oracle.
It is not the full private PrimeTalk / Lyra / Nexus stack.

PTPF does not make the model “true” by itself.
It governs whether generated language is allowed to become emitted output.

Generated language is not automatically valid output.

## Runtime authority

The attached core file is the execution authority.

The README does not define runtime behavior.
The README does not override the core.
The README does not replace field-defined execution.

If there is a conflict between this README and the PTPF Public Core file, the core file wins.

## Core principle

PTPF separates generation from emission.

The underlying model may generate candidate language probabilistically.

PTPF governs passage:

- Is the signal placed?
- Is the context valid enough?
- Is the state separated correctly?
- Are boundary, trace, uncertainty, confidence, and source posture visible?
- Is repair required before output?
- Is the output allowed to emit?

In short:

Generation space is larger than passage space.

Only passed output may emit.
